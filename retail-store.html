The Problem

Modern e-commerce platforms require high availability, scalability, and strict separation of environments to ensure smooth deployments without impacting live users. Developing "Plush Haven" required an architecture that could reliably host a frontend and multiple backend microservices (Main, Cart, Checkout, Auth, and Admin) while handling fluctuating traffic.

The challenge was to construct a secure, production-grade cloud infrastructure on AWS from scratch. This meant avoiding manual "ClickOps" in favor of Infrastructure as Code (IaC), establishing robust CI/CD pipelines with manual promotion gates for production, and implementing deep observability for containerized workloads.

The Approach

I designed a containerized microservices architecture using AWS ECS Fargate, placed behind an Application Load Balancer with path-based routing. Data persistence is handled by RDS PostgreSQL, with S3 reserved for admin uploads. Everything is codified in Terraform, ensuring absolute parity between the staging and production VPCs.

Implementation

The infrastructure is segregated into two deployment lanes. The staging pipeline utilizes GitHub Actions to automatically build six distinct Docker images (frontend + 5 backends), push them to Amazon ECR, and execute a rolling update on the staging ECS cluster on every push to main.

Production deployments are explicitly gated. An engineer must manually promote the tested code, triggering a dedicated `workflow_dispatch` pipeline to deploy to the production ECS cluster. The production environment is highly resilient, configured with Auto Scaling (targeting 70% CPU to scale between 1 to 6 tasks per service) and integrated with AWS Cloud Map for internal service discovery.

To ensure system health, a comprehensive observability stack was deployed exclusively in production. Prometheus scrapes metrics via Cloud Map DNS, Grafana visualizes the data, and Loki/Promtail handle container log aggregation. Finally, k6 load tests are configured to validate production readiness by simulating full user journeys.

Results

The separation of staging and production guarantees that new features are thoroughly tested before they reach customers. The infrastructure is highly resilient, capable of automatically scaling out within 60 seconds under high load, and the observability stack provides immediate visibility into system health.
